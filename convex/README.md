# 🛡️ Super Todo - Convex Backend

This directory contains the Convex backend functions for Super Todo.

## Files

- `schema.ts` - Database schema for projects and tasks
- `projects.ts` - Project queries and mutations
- `tasks.ts` - Task operations and AI auto-complete
- `notifications.ts` - Discord/Telegram webhook actions
- `daily.ts` - Scheduled morning/evening sync actions
- `testNotifications.ts` - Test functions for webhooks

## Environment Variables (Convex Dashboard)

Set these in your Convex Dashboard → Settings → Environment Variables:

### Discord
```
DISCORD_WEBHOOK=https://discord.com/api/webhooks/YOUR/WEBHOOK
```

### Telegram
```
BOT_TOKEN=your-bot-token-from-botfather
GROUP_ID=-1001234567890
```

⚠️ These are used by scheduled actions and server-side notifications.

## Scheduled Jobs (Crons)

Configure in Convex Dashboard:

1. Go to: https://dashboard.convex.dev/d/grand-sturgeon-25/schedules
2. Add scheduled functions:

### Morning Sync (8:00 AM UTC)
- **Function**: `daily.scheduledMorningSync`
- **Schedule**: `0 8 * * *` (cron format)

### Evening Report (8:00 PM UTC)  
- **Function**: `daily.scheduledEveningReport`
- **Schedule**: `0 20 * * *` (cron format)

## Testing Notifications

### Test from Convex Dashboard:

1. Go to: https://dashboard.convex.dev/d/grand-sturgeon-25/functions
2. Run `testNotifications:testAll` with args: `{ "message": "Hello from Super Todo!" }`

Or test individually:
- `testNotifications:testDiscord` - Test Discord only
- `testNotifications:testTelegram` - Test Telegram only
- `testNotifications:checkEnv` - Check if env vars are set

### Test from CLI:

```bash
# Check env vars
bunx convex run testNotifications:checkEnv

# Send test notification
bunx convex run testNotifications:testAll -- '{"message": "Test!"}'
```

## Schema

### projects
- `name`: string
- `group`: "important" | "hobbies"
- `status`: "active" | "archived"
- `lastWorkedOn`: number (timestamp)

### tasks
- `projectId`: Id<"projects">
- `text`: string
- `completed`: boolean
- `completedAt`: optional number
- `autoGenerated`: boolean
- `createdAt`: number
